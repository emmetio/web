Markup abbreviations

Emmet’s markup abbreviations looks like a regular code snippets available in most modern text editors. In editors, code snippets are static: you must predefine a short name which will be expanded into larger code. For example, div → <div class=""></div>

Emmet uses dynamic expressions: type anything and it will be transformed into full code. CSS selectors.

Abbreviation capturing

Due to dynamic nature of Emmet abbreviations, almost any word can be a snippet. Use of Tab to expand abbreviation key may lead to unexpected behavior if you want to use editor’s native snippets or put tab character after a word. To overcome this problem, Emmet uses abbreviation capturing: when you start typing something that looks likes abbreviation, Emmet adds underline indicating that expression can be expanded. When abbreviation becomes complex (e.g. it contains attributes or multiple elements), a popup with preview of expanded result is displayed.

After abbreviation was captured, hit Tab key inside abbreviation to expand it, Esc key to remove capturing.

To expand abbreviations with Tab key. Press Esc key to reset capturing.

Tag commenting

If enabled, adds comment with tag details before opening and/or after closing tag of expanded element which contains attribute triggers. A trigger is simply an attribute that should be present in element in order to add comments.

Tag details are described with templates: a string with fields between [ and ]. A field may contain arbitrary characters and upper-cased attribute name: entire field will be displayed only if specified attribute is present in element and attribute name will be replaced with actual attribute value.

Use \n to add a newline.
