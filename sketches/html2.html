Markup abbreviations

Emmet’s markup abbreviations looks like a regular code snippets available in most modern text editors. In editors, code snippets are static: you must predefine a short name which will be expanded into larger code. For example, divc → <div class=""></div>

Emmet uses dynamic expressions: type anything and it will be transformed into full code. Emmet abbreviations looks similar to CSS selectors so it should be very familiar to most front-end developers. Read more about abbreviation syntax.

JSX support

In JavaScript files, all camel-cased class names are treated as tag names: UI.MyComponent.foo will expand to <UI.MyComponent className="foo"></UI.MyComponent> instead of <UI class="MyComponent foo"></UI> in HTML. Emmet follows JSX rules and outputs className and htmlFor instead of class and for attributes.

Captures Emmet abbreviation as you type text (displayed with underline). Captured abbreviation can be expanded by Tab key if caret is inside it. Available in limited syntaxes only: HTML, XML, CSS, SASS etc.

Displays popup with preview of captured complex abbreviation: with attributes or multiple elements.

If enabled, adds comment with tag details before opening and/or after closing tag of expanded abbreviation element which contains attribute triggers. A trigger is simply an attribute that should be present in element in order to add comments.

Tag details are described with templates: a string with fields between [ and ]. A field may contain arbitrary characters and upper-cased attribute name: entire field will be displayed only if specified attribute is present in element and attribute name will be replaced with actual attribute value.

Use \n to add a newline.

When enabled, Emmet will treat class names starting with - as element and with _ as modifier in BEM notation. These class names will inherit block name from current or ancestor element. For example, the abbreviation ul.nav.nav_secondary>li.nav__item can be shortened to ul.nav._secondary>li.-item with this option enabled.

JSX support

In JSX (or JavaScript, TypeScript etc.), all Emmet abbreviations should be prefixed. A prefix is simply a character (or multiple characters) that must precede abbreviation in order to expand it. Without explicit prefix, Emmet would match abbreviations for every variable name or property access, which is just annoying.

All camel-cased class names are treated as tag names: UI.MyComponent.foo will expand to <UI.MyComponent className="foo"></UI.MyComponent> instead of <UI class="MyComponent foo"></UI> in HTML. Emmet follows JSX rules and outputs className and htmlFor instead of class and for attributes.

Empty value disables prefix and abbreviations will be captured and expanded the same as in HTML

Snippets

  • Markup snippets are just shortcuts for another Emmet abbreviations.
  • Shape-up default element attributes and their order: a → a[href]. If you expand a abbreviation, it will output <a> element with empty href attribute.
  • Snippet can out output multiple elements: nav → ul.nav>li.nav-item*4>a.
  • To output arbitrary content as snippet value, use text expressions: c → {<-- ${0} -->}.
  • Use ${N} to specify tabstops, where N is an integer. By default, Emmet automatically generates tabstops for empty attributes and elements, you can put a tabstop inside attribute value or text expression to explicitly traverse between them with Tab key: ah → a[href="http://${1}"]. You can add a placeholder to a tabstop: ${1:some text}. Placeholder content will be outputted and selected on focus.
